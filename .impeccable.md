## Design Context

### Users
Primary users are web developers and vibe coders who want to inspect React component hierarchy, structure, and organization directly inside running apps.
They use react-reinspect as an in-context debugging aid to understand UI composition faster, validate behavior, and iterate without switching tools constantly.

### Brand Personality
Professional, innovative, free.
The interface should feel like a serious frontend engineering tool: precise, reliable, and confident, while staying approachable and low-friction for fast iteration.

### Aesthetic Direction
Build a polished developer-tool aesthetic that emphasizes precision and clarity over decoration.
Support both light and dark themes as first-class experiences.
No explicit visual references or anti-references were provided, so direction should stay modern, restrained, and intentionally technical without becoming generic.

### Design Principles
1. Prioritize information clarity and precision: hierarchy, labels, controls, and state indicators must be immediately legible.
2. Design for speed of debugging: reduce interaction friction, keep actions contextual, and preserve smooth real-time feedback.
3. Ship professional-grade fit and finish: consistent spacing, typography, interaction states, and visual rhythm across the UI.
4. Maintain dual-theme quality: light and dark should both feel intentionally designed, not inverted afterthoughts.
5. Enforce high accessibility by default: target strong contrast, keyboard navigability, clear focus states, semantic controls, and reduced-motion respect.
